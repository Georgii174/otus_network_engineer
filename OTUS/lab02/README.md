## Выполнение ДЗ

### Начнем с самого сложного (5.План работы и изменения зафиксированы в документации.)

####
Настроить политику маршрутизации (Policy-Based Routing) для сетей офиса Чокурдах с целью:
- Распределения трафика клиентов между двумя провайдерскими линками (R25 и R26)
- Обеспечения отказоустойчивости при сбое одного из линков
- Автоматического переключения трафика на резервный канал с помощью IP SLA

Настройка интерфейсов R28

~~~! Отслеживание доступности R26
ip sla 1
 icmp-echo 45.0.2.1 source-interface Ethernet0/0
 frequency 5
ip sla schedule 1 life forever start-time now

! Отслеживание доступности R25
ip sla 2
 icmp-echo 32.0.5.1 source-interface Ethernet0/1
 frequency 5
ip sla schedule 2 life forever start-time now

Принцип работы:
Каждые 5 секунд отправляется ICMP-запрос (ping) на адрес соседнего маршрутизатора.
Если ответ не получен — линк считается недоступным.
~~~

Настройка Track-объектов
~~~
track 10 ip sla 1 reachability
 delay down 10 up 5

track 20 ip sla 2 reachability
 delay down 10 up 5

Параметры:

delay down 10 — задержка 10 секунд перед переключением (защита от флаппинга)

delay up 5 — задержка 5 секунд перед возвратом на основной линк
~~~
Настройка маршрутов по умолчанию
~~~
Маршрут	    Next-Hop	    AD	Track	Роль
Основной	45.0.2.1 (R26)	1	10	    Активен при работающем линке 
ip route 0.0.0.0 0.0.0.0 45.0.2.1 track 10
Резервный	32.0.5.1 (R25)	10	20	    Активен при сбое основного
ip route 0.0.0.0 0.0.0.0 32.0.5.1 10 track 20
~~~
Настройка политики маршрутизации (PBR) 

ACL для определения трафика
~~~
ip access-list extended acl_vpc30
 permit ip host 192.168.20.10 any

ip access-list extended acl_vpc31
 permit ip host 192.168.20.30 any
~~~
Route-map для маршрутизации
~~~
route-map PBR_to_R26 permit 10
 match ip address acl_vpc30
 set ip next-hop verify-availability 45.0.2.1 1 track 10

route-map PBR_to_R25 permit 20
 match ip address acl_vpc31
 set ip next-hop verify-availability 32.0.5.1 1 track 20

Логика работы:
Правило	    Условие (match)	                Действие (set)	            Приоритет
permit 10	source = VPC30 (192.168.20.10)	next-hop = 45.0.2.1 (R26)	Высший
permit 20	source = VPC31 (192.168.20.30)	next-hop = 32.0.5.1 (R25)	Средний
~~~ 
Применение PBR на интерфейсе:
~~~
interface Ethernet0/2.250
 ip policy route-map PBR_to_R26
~~~
Настройка обратных маршрутов (R25 и R26)
Для корректного возврата трафика от клиентов Чокурдаха на R25 и R26 добавлены статические маршруты:
~~~
На R26:
ip route 192.168.20.0 255.255.255.224 45.0.2.2

На R25:
ip route 192.168.20.0 255.255.255.224 32.0.5.2
~~~
Настройка маршрута по умолчанию на R27 (Лабытнанги)
~~~
ip route 0.0.0.0 0.0.0.0 32.0.2.1
~~~
